# 更新日志

## v0.0.2

### 新功能

**成本追踪**
- 状态栏实时显示累计花费（`$ Cost` 指标）
- 可配置时间窗口：N 天 / 月 / 年 / 全部
- 按项目白名单过滤计费
- 内置 11 个常用模型价格（Claude、GPT、Gemini、DeepSeek）

**价格同步**
- 支持 3 个同步源：LiteLLM、models.dev、BaseLLM
- 下拉框选择同步渠道（单个或全部）
- 用户手动设置的价格（source: "manual"）不会被同步覆盖
- 支持通过输入框新建自定义模型价格

**Codex CLI 支持**
- 监控 OpenAI Codex CLI 会话（`~/.codex/sessions/`）
- Claude Code / Codex CLI 可独立开关
- 两种来源使用统一的项目路径编码

**数据管理**
- 按模型删除历史数据（带确认）
- 删除全部数据（两步确认）
- 删除后模型列表自动刷新

### Bug 修复

- **修复成本计算**：解析器现在计算每轮 token 增量，而非存储累计会话总量。消除了严重的重复计费。
- **修复重复条目**：相同 token 数的重复 JSONL 行现在会被去重（零增量跳过）。
- **修复图表 X 轴**：午夜现在显示为 "0:00" 而非 "24:00"。
- **修复删除按钮**：用内联确认 UI 替换了 `window.confirm()`（在 Tauri webview 中无法弹出）。
- **修复模型列表刷新**：删除操作后下拉框会更新。
- **修复新建模型按钮**：当模型已存在于同步数据（但未显示）时不再静默失败。

### 改进

- 弹出窗口现在可以通过拖动空白区域移动
- 项目白名单支持正常路径格式（`/Users/name/project`），无需手动编码
- 成本时间窗口支持数字输入（如 "3 天" 而非仅 "天"）
- 项目白名单 placeholder 展示格式示例
- 价格表仅显示有实际使用数据的模型 + 手动添加的模型
- 同步渠道偏好持久化到配置
- `CostConfig::cost_since()` 方法消除重复计算逻辑
- 移除轮询热路径中不必要的 `to_string()` 分配

## v0.0.1

初始版本。

- macOS 状态栏实时显示输出速率、输入速率、TTFT
- 多模型追踪，图表按颜色区分
- 交互式图表，支持 1 小时 / 今天 / 昨天时间范围
- 可配置显示模式（最近一次请求 / 滚动平均）
- 模型别名和白名单过滤
- 深色 / 浅色 / 跟随系统主题
- SQLite 本地存储，WAL 模式
